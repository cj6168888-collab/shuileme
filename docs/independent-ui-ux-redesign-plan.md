# 睡了么 UI/UX 重新设计总方案

版本：V1.0  
原则：所有正式视觉素材都由绘图模型按用途单独设计，不再从整张概念图或截图里裁切。

## 1. 纠偏结论

现有实现最大问题不是功能方向，而是视觉生产方式错了：参考图只能作为风格标尺，不能作为素材仓库。正式 APK 必须建立独立资产库，每个页面背景、角色、按钮、图标、状态插画、动效帧都要单独生成，保证清晰度、构图、透明边缘、点击组件和无障碍文本都能被原生 Android 正确承载。

旧规则作废：

- 不再直接裁切整张设计图当 App 页面素材。
- 不再把按钮、文字和背景压进同一张大图。
- 不再让截图里的占位文案进入最终 APK。
- 不再用红色按钮承担普通聊天或普通提示。
- 不再让小助手聊天页像设置页或状态页。

新规则：

- 参考图只定义气质：温馨、可爱、3D、暖白、蓝绿橙红、大字体、大按钮。
- 组件必须由原生 View 承担点击、文字、状态和无障碍；图片只负责视觉资产。
- 资产按页面和组件单独生成，保留提示词、用途、尺寸和验收截图。
- 小助手页优先做“面对面活人陪伴感”，减少卡片和说明文字。
- 强唤醒页最高优先级是醒来确认安全，视觉必须直接、醒目、少干扰。

## 2. 品牌视觉系统

### 2.1 品牌关键词

- 温馨可信
- 明快亲切
- 家人陪伴
- 中老年友好
- 非医疗诊断
- 夜里安静、危急时清楚

### 2.2 吉祥物

主吉祥物是“狗熊”：圆润、柔软、闭眼安心、像家里床头小伙伴。它不是医生、不是警报器，而是“安静守着你的人”。

状态形象：

- 睡前：狗熊盖被子、抱月亮，表达“安心入睡”。
- 守护中：狗熊轻轻趴在床边或月亮旁，表达“我在”。
- 早安：狗熊挥手、阳光从窗边照进来。
- 强唤醒：狗熊退居辅助，主视觉换成红色闹钟和高亮背景。
- 空状态：狗熊抱着小盾牌或小枕头，表达“准备好了”。

### 2.3 色彩

主色必须明快，不做沉重暗黑医疗风。

| 用途 | 颜色 | 说明 |
| --- | --- | --- |
| 背景暖白 | `#FFF8EA` | 主背景，柔和不刺眼 |
| 主按钮蓝 | `#2F6FEA` | 开始守护、确认、主要行动 |
| 安全绿 | `#4CAF67` | 已准备、我没事、自动取消 |
| 晨光橙 | `#FF9D22` | 早安、提醒、轻建议 |
| 强唤醒红 | `#F24A3D` | 只用于强唤醒和紧急升级 |
| 深蓝标题 | `#16325C` | 品牌标题、首页主标题 |
| 正文灰 | `#4E5969` | 普通说明 |

使用限制：

- 红色只用于异常、强唤醒、紧急通知，不用于普通聊天按钮。
- 深蓝只做标题和夜间守护层，不做大面积沉闷背景。
- 卡片底色保持暖白、浅绿、浅橙、浅蓝，不做单一色系刷屏。

### 2.4 字体与按钮

面向中老年用户，核心文字要大。

| 元素 | 字号 |
| --- | --- |
| 首页标题 | 30sp |
| 页面标题 | 26sp |
| 主按钮 | 24sp |
| 卡片标题 | 21sp |
| 正文 | 18sp |
| 次要说明 | 16sp |
| 强唤醒标题 | 40sp |

按钮规则：

- 主按钮高 76-88dp。
- 强唤醒主按钮高 96dp 以上。
- 圆角 22-30dp，接近参考图的柔软胶囊风格。
- 按钮可以有独立位图质感，但文字和点击态必须是原生组件。
- 按钮内左侧优先用图标：播放、盾牌、太阳、心、闹钟、对勾。

## 3. 信息架构

底部只保留三个主入口：

1. 守护
2. 早安
3. 我的

小助手不是单独底部入口，而是贯穿三个页面：

- 守护页：睡前检查、守护开始、异常说明。
- 早安页：问候、睡眠汇报、喝水吃药。
- 我的页：联系人、声音、蓝牙、小助手偏好。
- 聊天页：从头像或按钮进入，进入后直接可说话。

## 4. 核心页面设计

### 4.1 启动与首页

目标：打开即懂，第一屏就像参考图左侧主手机。

布局：

- 顶部：品牌名“睡了么”，右侧小助手小头像。
- 中段：独立生成的夜间卧室狗熊场景，不能再用截图裁切。
- 状态浮层：盾牌图标 + “守护已就绪”或“还差 1 项设置”。
- 主按钮：蓝色大按钮“开始守护”，左侧播放图标。
- 底部：3 个检查项，最多一行显示：麦克风、音量、联系人。

交互：

- 点“开始守护”后进入睡前确认，再启动守护。
- 点小助手头像进入实时聊天，小助手自动开口，不要求点发送。
- 权限缺失时，主按钮仍可点，点击后只处理真正影响本次守护的权限。

### 4.2 睡前确认

目标：不做问卷，用 20 秒完成。

页面内容：

- 大字：“今晚我会静静守着。”
- 3 项检查：麦克风、音量/蓝牙、紧急联系人。
- 两个按钮：“开始守护”“先去处理”。

这里不展示复杂检测算法、不展示波形、不展示健康焦虑文案。

### 4.3 守护中

目标：夜里不打扰。

视觉：

- 夜间深蓝渐变背景，但按钮和文字仍清楚。
- 狗熊睡觉小动画或静态图，动画幅度极小。
- 状态文案：“正在守护，安心睡吧。”
- 大按钮：“停止守护”。

禁止：

- 不发运行提示音。
- 不定时播报“我还在运行”。
- 不显示复杂声波图刺激用户。

### 4.4 强唤醒

目标：半睡半醒 3 秒内知道要做什么。

视觉：

- 独立生成晨光/警醒背景，不用恐怖画面。
- 独立生成红色闹钟 3D 插画。
- 标题：“请醒一下”或“起床确认一下”。
- 主按钮：红色或高亮绿色大按钮“我没事”。
- 次按钮：“暂停 10 分钟”“结束本晚”。

交互：

- 支持语音：“我没事”停止唤醒。
- 倒计时显示：“还没有确认，60 秒后通知联系人。”
- 倒计时期间先播亲人录音，再播铃声和震动。

文案限制：

- 不说“呼吸暂停诊断”。
- 不说“你有生命危险”。
- 只说“检测到高风险睡眠异常，请先确认安全”。

### 4.5 早安护理

目标：像参考图第二台手机，明亮、简单、可执行。

布局：

- 顶部：太阳图标 + “早安护理”。
- 中段：狗熊挥手晨间场景。
- 三张大卡：喝水、吃药、昨晚汇报。
- 底部按钮：我醒了 / 今天不提醒。

喝水卡：

- 绿色浅底，玻璃水杯独立图标。
- 按钮：“我喝了”“稍后提醒”。

吃药卡：

- 橙色浅底，胶囊或药盒独立图标。
- 只有用户设置后显示。
- 按钮：“已吃药”“15 分钟后提醒”“今天不提醒”。

睡眠报告卡：

- 一句话摘要，不做医学结论。
- 示例：“昨晚记录 4 次波动，3 次自动恢复。”

### 4.6 小助手实时聊天

目标：打开就能聊，像有人坐在面前听，不像表单。

交互范式对齐 Gemini Live：小助手页不是消息列表，也不是“语音转文字再发送”的屏幕流程，而是持续听、可打断、可看见、会即时安抚的 Live 陪伴舞台。真实实时模型接入前，旧式 ASR/TTS 文本链路只能作为兜底。

第一屏：

- 顶部不写“AI 聊天”，写助手名字或“我在听您说”。
- 中上部是独立生成的助手半身大形象，占屏幕 35%-45%。
- 助手有轻呼吸、眨眼、点头动效。
- 中部一个大语音气泡：显示助手正在说的话。
- 底部三个动作：我来说、暂停、看一眼。

第一次认识：

- 不做单独认识页。
- 小助手第一句话：“您好呀，我先陪着您。您想给我起个什么名字？我该怎么称呼您？”
- 用户可直接说：“你叫暖暖，像女儿一样陪我，叫我奶奶。”
- 系统提取：助手名、身份偏好、主人称呼、关怀风格。

语音：

- 进入聊天页自动听。
- 用户说完自动提交。
- 助手说话时可打断，但按钮不能用红色。
- 按钮文案用“我来说”，颜色蓝或绿。

摄像头：

- 安装或首次聊天可解释用途，但不制造恐惧。
- 聊天中自动低清采样时要有可见提示：“我看一眼周围，帮您记东西。”
- 重点记忆：钥匙、手机、药盒、眼镜、钱包、遥控器、水杯、证件。
- 找东西检索时先说：“您别急，我想想，先陪着您。”

### 4.7 我的

目标：设置少、字大、能让家人帮忙。

分区：

- 我的助手：角色、名字、语气偏好。
- 唤醒声音：内置音乐、手机歌曲、亲人录音。
- 蓝牙音箱：测试音、音量上限、断连回退。
- 紧急联系人：电话/短信、等待时间、测试模式。
- 隐私与联网：DeepSeek Key、本机数据、清空记录。

设置页不要一屏堆满开关。每个卡片只显示当前状态和一个“去设置”。

## 5. 组件设计

### 5.1 主按钮

资产拆分：

- 蓝色开始守护按钮底图。
- 绿色我没事按钮底图。
- 橙色早安护理按钮底图。
- 红色强唤醒按钮底图。
- 圆形播放、盾牌、太阳、心形、闹钟图标。

原生实现：

- 背景图只做质感。
- 文字和图标可由 Android View 叠加，保证大字体不糊。
- 按下态加轻微缩放和阴影变化。

### 5.2 卡片

卡片类型：

- 状态卡：盾牌、对勾、准备好。
- 护理卡：水杯、药、晨练、天气。
- 报告卡：昨晚总结、自动取消、强唤醒。
- 设置卡：联系人、声音、蓝牙、隐私。

卡片规则：

- 圆角 20dp。
- 内边距 18-22dp。
- 卡片高度不低于 88dp。
- 标题 21sp，正文 18sp。
- 图标独立生成，不从场景里裁。

### 5.3 底部导航

图标独立生成：

- 守护：小盾牌。
- 早安：小太阳。
- 我的：小人头像。

状态：

- 选中：蓝色或橙色，不使用红色。
- 未选中：灰蓝。
- 文案大于 14sp，图标 26dp。

### 5.4 小助手动效

第一版用帧动画或简单 View 动画：

- breathing：轻微上下浮动。
- blink：眼睛帧切换。
- nod：回答时点头。
- listen：手边小波纹，不做复杂音频波形。
- think：轻微抬头 + 气泡“我想想”。
- reassure：微笑 + 轻挥手。

每个角色至少 6 个独立姿态图，不从同一张角色卡裁切。

## 6. 权限 UX

权限不在启动时一股脑弹出。按场景解释：

- 麦克风：开始守护前解释。
- 通知：要保证夜间异常能叫醒。
- 电话/短信：只在用户开启紧急联系人时请求。
- 摄像头：第一次进入小助手视觉陪伴时解释。
- 蓝牙：使用系统音频路由，不把用户困在复杂连接页。

文案必须短：

- “为了夜里能听到异常声音，需要麦克风。”
- “只有唤醒失败时，才会通知你指定的人。”
- “小助手看一眼，是为了帮您找东西、读字、记药盒位置。”

## 7. 离线与联网边界

当前产品方向已经从纯离线转为联网和 AI 优先，但安全底线不能依赖网络：

- 聊天、建议、看图、情绪陪伴：联网 AI 优先。
- 夜间守护、唤醒、倒计时、紧急联系人：本地兜底必须可用。
- 不把 API Key 打进 APK。
- 不默认上传整夜录音、亲人录音、联系人电话。

## 8. 验收标准

视觉验收：

- 首页、早安、强唤醒、小助手聊天四屏都不能出现截图裁切痕迹。
- 所有角色和场景的光照、材质、圆润程度一致。
- 大字体模式下按钮文字不溢出。
- 红色只出现在强唤醒或紧急场景。
- 小助手聊天页第一屏看起来像“面对面陪伴”，不是设置页。

交互验收：

- 新用户 60 秒内能开始守护。
- 打开聊天页不用点发送即可说话。
- 助手说话时用户能打断。
- 找东西时先有安抚语，再检索记忆。
- 强唤醒 3 秒内可点到“我没事”。

工程验收：

- 每张新资产都有独立文件名、用途、提示词和尺寸。
- 每个页面至少保留一张模拟器截图。
- `test.ps1` 通过。
- APK 可安装启动。

## 9. 实施顺序

第一批先重做 12 个关键资产：

1. App 图标狗熊抱月亮。
2. 首页夜间卧室背景。
3. 首页睡觉狗熊主体。
4. 蓝色开始守护按钮质感。
5. 早安护理背景。
6. 早安挥手狗熊主体。
7. 红色闹钟强唤醒主体。
8. 强唤醒暖光背景。
9. 温柔姐姐聊天半身形象。
10. 贴心小妹聊天半身形象。
11. 懂事小弟聊天半身形象。
12. 阳光小哥聊天半身形象。

第二批补组件：

- 水杯、胶囊、盾牌、心形、太阳、联系人、蓝牙音箱、麦克风、摄像头、钥匙、眼镜、手机、药盒。
- 四个角色的听、说、想、点头、挥手、安抚姿态。
- 唤醒音乐、亲人录音、紧急通知插画。

第三批做原生 UI 重构：

- 首页换成真实组件 + 独立背景。
- 早安页换成护理卡片。
- 强唤醒页换成独立闹钟和大按钮。
- 小助手页重构为面对面实时语音界面。
- 设置页收敛成大卡片入口。
