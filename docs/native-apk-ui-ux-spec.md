# 狗熊睡眠原生 APK UI/UX 设计规范

目标：原生 Android APK，面向普通家庭用户，尤其照顾中老年人。界面要简单明朗，大字体，大按钮，色泽明快亲切，夜里半睡半醒也能看懂、点准、停止。

## 1. 设计原则

- 开箱即用：不注册、不登录、不收费、不接服务器，首页直接能开始守护。
- 大字优先：核心信息不用小字解释，标题、状态、按钮都要适合中老年人阅读。
- 少选择：默认给“标准模式”，高级设置放后面，不让用户第一次就被参数淹没。
- 明确状态：用户随时知道当前是“未开始 / 守护中 / 已暂停 / 权限不完整 / 昨晚已完成”。
- 夜间可操作：强唤醒页必须大按钮、高对比、少文字，支持语音、音量键、电源键辅助确认。
- 亲切但不幼稚：颜色明快温暖，不做医院感，也不做儿童化。
- 本地可信：所有隐私、录音、联系人都明确显示“仅保存在本机”。

## 2. 视觉风格

### 2.1 色彩

主色建议：

- 主色：清晨蓝 `#2F80ED`，用于主按钮、守护状态、关键操作。
- 辅色：薄荷绿 `#27AE60`，用于安全、已完成、状态正常。
- 温暖色：晨光橙 `#F2994A`，用于提醒、建议、轻唤醒。
- 高风险：柔和红 `#EB5757`，用于强唤醒、紧急联系人升级。
- 背景：暖白 `#FFFDF8`，比纯白更亲切。
- 文本：深灰 `#243040`，避免纯黑刺眼。
- 次级文本：灰蓝 `#667085`。

使用规则：

- 首页以暖白背景 + 清晨蓝主按钮为主。
- 风险页才使用红色，不把红色用于普通提醒。
- 早晨报告可用蓝、绿、橙区分“记录、自动取消、建议”。
- 夜间强唤醒使用深色背景 + 大号白字 + 红/橙按钮，保证看得清。

### 2.2 字体

Android 使用系统字体，支持用户系统字体缩放。

建议字号：

- 首页状态标题：28-32sp。
- 主按钮文字：22-24sp。
- 页面标题：24-28sp。
- 卡片标题：20-22sp。
- 正文：18sp。
- 次要说明：16sp，少用。
- 强唤醒页主文案：34-42sp。

不要使用 12-14sp 作为关键信息。所有核心操作都应在 18sp 以上。

### 2.3 按钮和点击区域

- 主按钮高度：72-88dp。
- 普通按钮高度：56-64dp。
- 图标按钮最小点击区域：48dp。
- 强唤醒确认按钮：全宽，至少 96dp 高。
- 按钮圆角：12-16dp，亲切但不过度圆润。
- 按钮文字不用长句，优先“开始守护”“我没事”“停止唤醒”。

## 3. 信息架构

底部导航建议 3 个主入口：

1. 守护
2. 记录
3. 设置

不要超过 3 个底部标签。中老年用户更适合固定入口，不适合复杂抽屉菜单。

## 4. 核心页面

### 4.1 首次引导

引导页要尽量短。默认提供“直接开始”，让用户不填任何资料也能使用标准模式。完整引导最多 3 步，更多细项放到设置页。

0. 欢迎
   - 标题：狗熊睡眠，守护你的夜晚
   - 说明：本机识别异常声音和动作，必要时唤醒你。
   - 主按钮：直接开始
   - 次按钮：简单设置

“直接开始”进入首页，使用标准模式，并在首页做睡前检查。

“简单设置”只走 3 步：

1. 你最担心什么？
   - 大选项：打鼾/喘息、噩梦惊醒、夜里动作大、先用默认
   - 单选即可，不用解释太多

2. 隐私
   - 标题：数据只在手机里
   - 说明：默认不保存整夜录音，只保存异常片段；也可以关闭片段保存。
   - 选项：保存异常片段 / 只保存时间线
   - 默认：保存异常片段

3. 唤醒方式
   - 大选项：只在高风险叫醒 / 鼾声先轻提醒 / 尽量少打扰
   - 可选：现在设置紧急联系人 / 以后再说

引导结束后进入睡前测试：

- 检查麦克风、通知、音量、蓝牙音箱、电量、紧急联系人。
- 用清晰图标显示通过/需处理。

引导设计禁区：

- 不要把健康问题做成长问卷。
- 不要在首次引导里展示策略矩阵。
- 不要要求用户先设置蓝牙、录音、联系人才能开始。
- 不要一次性请求所有权限，按功能触发。

### 4.2 首页：守护

首页只有一个核心动作。

布局：

- 顶部：问候 + 当前状态
- 中部：大圆形或大矩形主按钮
- 下方：今晚检查清单
- 底部：昨晚摘要

示例：

```text
今晚准备好了吗？
标准模式

[ 开始守护 ]

今晚检查
✓ 麦克风可用
✓ 音量正常
✓ 蓝牙音箱：卧室音箱
! 未连接紧急联系人

昨晚：记录 5 次，自动取消 3 次，强唤醒 0 次
```

状态文案：

- 未开始：今晚准备好了吗？
- 守护中：正在守护，安心睡吧
- 权限缺失：还差 1 项设置
- 昨晚完成：昨晚记录已生成

### 4.3 守护中页面

夜间启动后，界面应暗下来但保持可读。

显示：

- 正在守护
- 已运行时长
- 当前模式
- 输出设备
- 大按钮：停止守护

不显示复杂波形，避免用户焦虑。

### 4.4 强唤醒页面

这是最重要的页面。

要求：

- 锁屏上可见或通过高优先级通知拉起。
- 深色背景。
- 超大文字。
- 一个最主要按钮。
- 支持语音停止。

示例：

```text
请醒一下

检测到高风险睡眠异常
如果你没事，请说“我没事”

[ 我没事，继续守护 ]

[ 暂停 10 分钟 ]   [ 结束本晚 ]
```

如果即将通知联系人：

```text
还没有收到确认

15 秒后将通知紧急联系人

[ 我没事，不通知 ]
```

强唤醒页不能放“诊断”“呼吸暂停”等医学结论，只说“高风险睡眠异常”。

### 4.5 轻提醒页面

轻提醒通常不需要亮屏展示。若用户拿起手机，展示：

- 已轻提醒
- 声音和动作正在观察
- 如果恢复平稳，将自动取消

按钮：

- 我没事
- 以后这种情况只记录

### 4.6 早晨报告

早晨报告不要做复杂医学图表，按事件时间线组织。

顶部摘要：

- 睡眠时长
- 异常记录次数
- 自动取消次数
- 强唤醒次数
- 建议条数

事件卡片：

- 时间
- 事件类型
- 动作：只记录 / 轻提醒 / 自动取消 / 强唤醒 / 通知联系人
- 触发依据：声音、动作、可穿戴数据
- 用户反馈按钮：真实 / 误报 / 不确定

建议卡：

- “今晚可以尝试”
- 每条建议不超过两行
- 必须带依据，例如“因为昨晚 02:10 连续鼾声 8 分钟”

### 4.6.1 晨间护理页

用户点击“我醒了”或系统判断可能起床后进入。

页面结构：

- 顶部 3D 护理助手头像。
- 大字问候：“早安，昨晚辛苦了。”
- 三个大卡片：睡眠汇报、喝水、吃药。
- 一个语音按钮：和我聊聊。

吃药卡片：

- 只有用户设置后显示。
- 按钮：已吃药 / 15 分钟后提醒 / 今天不提醒。
- 未确认时，按设置间隔重复提醒。

晨练小贴士：

- 一次只给 1 条，例如“先坐起 1 分钟，再慢慢站起来”。
- 不给复杂运动处方。

天气小贴士：

- 离线默认显示“出门前记得查看天气”。
- 用户开启联网后，再显示真实天气。

大模型助手入口：

- 明确“可选联网增强”。
- 默认不上传整夜录音。
- 推荐用语音聊天，减少阅读负担。

### 4.7 唤醒声音页面

三个标签：

- 内置音乐
- 我的歌曲
- 亲人录音

每个音源条目：

- 名称
- 播放/暂停图标
- 设为轻提醒
- 设为强唤醒

亲人录音流程：

1. 点击“录一段亲人声音”。
2. 展示建议文案。
3. 按住录音或点击开始/停止。
4. 自动播放试听。
5. 保存并命名。

文案示例：

- “醒一下，确认你没事。”
- “慢慢呼吸，我在这里。”
- “听到了就说我没事。”

### 4.8 蓝牙音箱页面

目标是简单，不做复杂蓝牙管理器。

显示：

- 当前输出：手机扬声器 / 蓝牙音箱名称
- 按钮：打开系统蓝牙设置
- 按钮：播放测试音
- 开关：只在高风险时使用蓝牙音箱
- 音量上限滑块

如果断连：

“蓝牙音箱未连接。强唤醒会自动使用手机扬声器和震动。”

### 4.9 紧急联系人页面

默认关闭。

开启后流程：

1. 说明用途。
2. 添加联系人。
3. 选择通知方式：电话 / 短信 / 电话 + 短信。
4. 设置等待时间：60 秒 / 90 秒 / 120 秒。
5. 测试模式。

页面文案：

“只有在高风险唤醒失败时，才会通知你指定的人。”

联系人卡片：

- 姓名
- 电话号码后四位
- 通知方式
- 顺序
- 测试按钮

## 5. 组件规则

### 5.1 状态检查项

用清晰图标和文字：

- 正常：绿色对勾 + “已准备好”
- 需要处理：橙色感叹号 + “去设置”
- 不可用：红色叉 + “无法使用”

### 5.2 策略选择

使用三段式大按钮：

- 安静
- 标准
- 敏感

默认标准。不要让用户一上来配置十几个阈值。

### 5.3 风险标签

- 只记录：蓝色
- 自动取消：绿色
- 轻提醒：橙色
- 强唤醒：红色
- 已通知联系人：红色加电话图标

## 6. 中老年适配

必须做到：

- 全 App 支持系统大字体。
- 核心按钮不因字体放大而文字溢出。
- 页面不依赖小图标表达含义，图标旁必须有文字。
- 不使用长段落说明，分成短句。
- 重要操作二次确认，但不做复杂弹窗。
- 弱网或断网不影响任何核心功能。
- 颜色不能只靠颜色区分，还要有文字和图标。

建议加入：

- “长辈模式”：字号整体更大，首页只显示开始守护、停止守护、昨晚报告。
- “家人帮我设置”：允许家人一次性配置联系人、声音、蓝牙音箱。

## 7. 原生实现建议

- 技术栈：首个可安装 APK 使用 Java 原生 View 和 Android SDK 直编；后续产品化版本可迁移 Kotlin + Jetpack Compose + Material 3。
- 数据：Room + 本地加密。
- 音频：Android AudioRecord / MediaPlayer / AudioTrack。
- 蓝牙：优先使用系统音频路由，不自建复杂连接协议。
- 权限：按功能触发请求，不在启动时一次性请求。
- 通知：前台服务通知 + 高优先级唤醒通知。
- 强唤醒：全屏 Intent 或等效高优先级交互方案，需适配不同 Android 版本限制。

## 8. UI 验收标准

- 新用户 60 秒内能开始守护。
- 首页只有一个最高优先级动作：开始守护。
- 主要文字在中等字体缩放下不小于 18sp。
- Android 字体放大到 130%-150% 时，主流程不遮挡、不溢出。
- 强唤醒页半睡状态下 3 秒内能看懂并点击“我没事”。
- 断网状态下所有页面可用。
- 电话/短信权限被拒绝时，紧急联系人页面能解释回退方式。
- 蓝牙音箱断连时，首页和睡前测试能明确提示会回退手机扬声器。
