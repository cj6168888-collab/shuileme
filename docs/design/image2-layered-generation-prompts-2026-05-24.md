## 输出规格

页面效果图：

```text
输出单张 1080x2400 竖版 Android 手机界面效果图，正面平铺，无手机外壳，无透视，无倾斜。界面内容必须完整、清晰、可读。
```

透明素材：

```text
输出 PNG 风格素材，透明背景，主体居中，边缘干净，无文字，无多余投影，无裁切。素材需要适合 Android drawable-nodpi 使用。
```

组件素材：

```text
输出单个 UI 组件 PNG，透明背景或纯白卡片背景，保留圆角、阴影、图标、状态文字和布局比例。不要放进手机框，不要拼贴多个不相关组件。
```

## 10. 执行顺序

1. 先生成首页真实效果图，确认风格完全贴合参考图。
2. 再生成首页分层素材：Hero 插画、波形线、波形背景、吃药图标、健康习惯图标。
3. 再生成小助理真实效果图和数字人透明半身像。
4. 再生成睡眠报告、睡前自检、吃药提醒、健康习惯页面。
5. 最后生成设置、能力检查、主人档案、早安简报。

每批生成后都要和参考图对比：布局、颜色、信息层级、文字、功能入口必须一致；不一致就重新生成，不进入 App。

## 11. 生成后验收清单

首页必须满足：

- 只出现睡眠守护、睡眠波形检测、睡眠报告、吃药提醒、健康习惯、底部导航。
- 不出现“小助手”独立卡片、不出现“家人电话”、不出现“故事”、不出现“助眠音”、不出现“拾音设置”。
- 波形模块必须是整排深蓝卡片，不是小卡片。
- 首页必须一屏接近放完，不能做成长信息流。
- 睡眠报告入口必须保留，但不要展开成大段报告。
- 吃药提醒和健康习惯必须是首页高频入口，不要藏进设置。

小助理页必须满足：

- 数字人是页面主体。
- 按住说话和结束对话在底部清晰可点。
- 语音/音量可视化可见。
- 页面一屏完成，不做滚动长页。

素材必须满足：

- 透明素材不能带文字、不能带手机框、不能带多余背景。
- 同一角色、同一小熊、同一品牌图形要保持一致。
- 图标必须在 48dp 到 72dp 尺寸下仍然清楚。
- 所有中文必须无错字、无乱码、无重叠。
- 如果 image2 生成文字不稳定，文字层改由 Android 原生 UI 绘制，图片只保留无字视觉元素。
