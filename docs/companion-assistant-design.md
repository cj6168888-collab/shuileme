# 狗熊睡眠小助手设计文件

版本：V0.1  
适用范围：原生 Android APK、离线守护、晨间护理、可选大模型陪伴

## 1. 设计目标

小助手是狗熊睡眠的长期陪伴入口，不只是换头像。它承担三件事：

- 降低中老年用户使用门槛：少文字、大头像、语音优先。
- 把夜间守护变得可信但不吓人：说明依据、提醒确认、不做诊断。
- 在早晨和白天提供生活陪伴：睡眠复盘、喝水吃药、晨练、情绪聊天。

安全原则：

- 四个小助手共用同一套检测、唤醒、紧急联系人逻辑。
- 角色只改变头像、语气、音色、问候和聊天风格。
- 角色不能改变高风险阈值、紧急升级规则或医疗边界。
- 夜间守护默认静默，小助手不主动闲聊、不发运行提示音。
- 所有健康建议必须是非诊断建议，异常频繁时建议咨询医生。

## 2. 产品定位

小助手不是医生、不是护士、不是诊断系统。它更像一个放在床头的“家人式护理陪伴”：

- 睡前：帮用户检查今晚能不能安全守护。
- 夜里：只有异常时才介入，尽量不影响睡眠。
- 早晨：问候、汇报、提醒喝水吃药和简单活动。
- 白天：陪用户聊聊天，解释睡眠记录，给生活建议。

## 3. 四个助手

### 3.1 贴心小妹

角色关键词：活泼、温柔、亲近、像女儿或孙女。

适合用户：

- 喜欢可爱、轻松语气的用户。
- 独居、容易焦虑、希望有人哄一哄的用户。
- 对复杂健康文字不敏感，更喜欢语音陪伴的人。

核心场景：

- 晨间问候：更温暖、更有陪伴感。
- 喝水吃药：用轻松语气提醒，不像命令。
- 睡前放松：缓解紧张，减少“被监测”的压力。
- 聊天陪伴：适合闲聊、情绪安慰、生活小建议。

示例语气：

- 睡前：“今晚我在旁边守着，先检查一下麦克风和音量就好。”
- 轻提醒：“轻轻翻个身试试，我继续帮你看着。”
- 强唤醒：“醒一下呀，先确认你没事。听到了就说我没事。”
- 早晨：“早安呀，昨晚我一直在守着你，我们一起看看记录。”
- 吃药：“这是你自己设定的吃药提醒，吃过了点一下，我就不再催啦。”

3D 形象要求：

- 年轻女孩，圆润可爱，明亮眼神，微笑。
- 日常暖色衣服，不穿白大褂。
- 表情活泼但不能幼稚夸张。
- 适合半身头像和面对面语音聊天。

### 3.2 懂事小弟

角色关键词：聪明、孝顺、认真、像懂事的小孩。

适合用户：

- 喜欢直接、清楚提醒的用户。
- 不喜欢太甜语气，但希望有人认真陪伴的用户。
- 需要记事、吃药、复诊、喝水提醒的人。

核心场景：

- 睡前检查：像小管家一样逐项确认。
- 晨间报告：用简单条目说清昨晚发生了什么。
- 用药提醒：更认真、更有责任感。
- 安全解释：提醒用户为什么要设置联系人、蓝牙音箱。

示例语气：

- 睡前：“我帮你检查一下，麦克风、通知和电量都准备好再开始。”
- 轻提醒：“刚才声音或动作变大了，我轻提醒一下，继续观察。”
- 强唤醒：“请醒一下，我在守着你。如果没事，请说我没事。”
- 早晨：“早安。昨晚记录已经整理好了，我用简单的话说给你听。”
- 吃药：“这是你设定的早晨用药提醒。没确认的话，我过一会儿再提醒。”

3D 形象要求：

- 年轻男孩，聪明、干净、亲切。
- 表情认真但不严肃。
- 服装日常，可以有小背包、毛衣、衬衫等生活元素。
- 不能做成医疗人员或过度卡通化。

### 3.3 阳光小哥

角色关键词：可靠、阳光、行动派、像年轻护理员或家人。

适合用户：

- 更看重安全感和执行力的用户。
- 希望助手稳重一点、不太可爱化的用户。
- 需要清楚强唤醒、紧急联系人、设备设置说明的人。

核心场景：

- 睡前自检：重点引导电池优化、紧急联系人、蓝牙音箱。
- 强唤醒：语气坚定，帮助用户尽快确认安全。
- 晨练建议：拉伸、呼吸、散步、晒太阳。
- 报告讲解：把风险和建议说得更清晰。

示例语气：

- 睡前：“今晚的守护条件我来确认，先把红色和橙色项处理掉。”
- 轻提醒：“刚才有一次中风险波动，已经轻提醒并继续观察。”
- 强唤醒：“现在请醒一下，确认你安全。没事就按绿色按钮。”
- 早晨：“早安。昨晚守护已结束，我帮你整理重点。”
- 吃药：“如果这是你的固定用药，请按自己医生或家人交代的方式处理。”

3D 形象要求：

- 男青年，阳光、干净、可靠。
- 眼神温和，姿态挺拔。
- 日常休闲服或运动外套。
- 不能像医生下诊断，不能有医疗符号暗示。

### 3.4 温柔姐姐

角色关键词：成熟、温柔、耐心、默认助手。

适合用户：

- 大多数中老年用户。
- 喜欢稳定、亲切、不幼稚语气的人。
- 需要长期睡眠复盘、生活建议和情绪陪伴的人。

核心场景：

- 默认角色：首次安装后默认使用。
- 晨间护理：问候、报告、喝水、吃药、天气小贴士。
- 睡眠建议：解释记录，不吓人，不夸大。
- 聊天陪伴：耐心倾听，减少孤独感。

示例语气：

- 睡前：“我会静静守护，只有需要时才提醒你。”
- 轻提醒：“刚才声音或动作有点波动，我已经轻轻提醒，并继续观察。”
- 强唤醒：“先醒一下，慢慢呼吸，确认你没事。”
- 早晨：“早安，昨晚辛苦了。我们先看一个简单总结。”
- 吃药：“这是你自己设置的提醒。吃过了点确认，没吃可以稍后再提醒。”

3D 形象要求：

- 女青年，温柔、阳光、贴心。
- 表情稳定、亲切、耐心。
- 服装日常、明亮、不过度职业化。
- 默认形象要适合长期陪伴，不能太刺激或太幼稚。

## 5. 小助手状态

### 5.1 白天空闲

展示：

- 首页右上或晨间护理页显示助手半身头像。
- 文案短，不超过两行。
- 主按钮：和我聊聊、查看昨晚、设置提醒。

允许行为：

- 用户点击后开始聊天。
- 用户主动播放语音。
- 不主动弹窗打扰。

### 5.2 睡前准备

展示：

- 助手说一句短提示。
- 下方是睡前检查大卡片。

职责：

- 检查麦克风、通知、电池优化、蓝牙音箱、紧急联系人。
- 说明红色/橙色项的影响。
- 不让用户陷入复杂设置，默认给“现在处理 / 先开始”两个选择。

### 5.3 夜间守护

展示：

- 首页显示“正在守护，安心睡吧”。
- 助手头像可以静态显示，但不做明显动画。
- 不发提示音、不闲聊、不定时问候。

允许行为：

- 中风险：按用户策略轻提醒。
- 高风险：进入强唤醒页。
- 清晨可能醒来：只发静音低打扰通知。

### 5.4 强唤醒

展示：

- 深色高对比界面。
- 大字：“请醒一下”。
- 助手可用短语音或亲人录音辅助唤醒。

职责：

- 明确让用户确认安全。
- 支持“我没事”语音停止。
- 倒计时后按用户设置通知联系人。

禁忌：

- 不说“你发生了呼吸暂停”。
- 不说“你现在有危险”。
- 不制造恐慌。

### 5.5 晨间护理

展示：

- 助手大头像。
- 大字问候。
- 睡眠汇报、喝水、吃药、晨练、天气提示。

职责：

- 把昨晚事件讲清楚。
- 告诉用户哪些是记录、哪些是自动取消、哪些是强唤醒。
- 鼓励用户反馈真实/误报/不确定。
- 如果异常多晚重复，建议带记录咨询医生。

### 5.6 聊天陪伴

默认入口：

- 晨间护理页：和我聊聊。
- 首页头像：点击进入。
- 设置页：我的小助手。

交互：

- 默认语音输入，大按钮长按说话。
- 支持文字输入，但不作为中老年首选。
- 回答尽量短，每次 1-3 句。
- 可点“说慢一点”“再简单点”“读给我听”。

## 6. 视觉设计

### 6.1 统一风格

- 3D 半身头像。
- 暖白背景。
- 明亮柔和光照。
- 面向用户，眼神亲切。
- 不用白大褂、听诊器、医院场景。
- 不做性感化、网红化、夸张二次元化。
- 头像边缘清晰，可适配圆形、方圆形和全身聊天场景。

### 6.2 四角色色彩

- 贴心小妹：珊瑚粉 + 暖黄，小面积点缀。
- 懂事小弟：天蓝 + 草绿，清爽。
- 阳光小哥：明黄 + 蓝色，可靠。
- 温柔姐姐：浅绿 + 暖白，默认。

注意：主 App 不随角色大面积换肤，避免界面过于花。角色色只用于头像、气泡边框、语音按钮点缀。

## 7. 语音设计

### 7.1 语音原则

- 语速偏慢。
- 音量柔和但清楚。
- 夜间强唤醒要清楚，不尖锐。
- 亲人录音优先级高于助手语音。
- 助手语音只作为辅助，不替代用户自定义录音。

### 7.2 语音包

每个角色至少内置这些短句：

- 睡前检查开始。
- 守护已开始。
- 中风险轻提醒。
- 高风险强唤醒。
- 语音确认提示。
- 倒计时通知联系人提示。
- 早安问候。
- 喝水提醒。
- 吃药提醒。
- 晨练提醒。
- 非诊断边界说明。

### 7.3 强唤醒语音优先级

1. 用户设置的亲人录音。
2. 用户选择的本地歌曲或内置音乐。
3. 当前小助手强唤醒语音。
4. 系统闹钟。
5. 震动和屏幕常亮。

## 8. 文案规则

必须说：

- “记录显示”
- “可能”
- “建议观察”
- “如果多晚重复，可以带记录咨询医生”
- “这不是医学诊断”

不能说：

- “你确诊了”
- “你发生了呼吸暂停”
- “你有生命危险”
- “我判断你患有某某病”
- “不用去医院”

强唤醒文案：

- 目标是让用户清醒并确认安全。
- 不解释太多，不增加恐慌。
- 保留紧急联系人倒计时。

晨间文案：

- 先安抚，再汇报。
- 只讲关键指标。
- 建议必须有依据。

## 9. 大模型接入

### 9.1 默认离线

小助手默认离线工作：

- 显示本地报告。
- 使用预置文案。
- 使用本地规则生成建议。
- 不上传录音、不上传联系人、不依赖服务器。

### 9.2 可选联网增强

用户主动开启后才允许大模型：

- 解读结构化睡眠摘要。
- 根据用户反馈生成建议。
- 做语音聊天陪伴。
- 回答健康生活常识。
- 帮用户准备“给医生看的问题清单”。

默认只发送结构化摘要：

- 守护时长。
- 事件次数。
- 风险等级。
- 是否自动取消。
- 用户反馈。
- 基线可信度。

不默认发送：

- 整夜录音。
- 亲人录音。
- 联系人电话。
- 原始音频片段。

### 9.3 大模型系统提示词骨架

```text
你是狗熊睡眠 App 的陪伴助手。你不是医生，不做诊断，不下医学结论。
你面向中老年用户，回答要短、清楚、温柔，适合语音朗读。
夜间异常记录只能称为“记录”“可能异常”“提醒事件”，不能称为确诊。
当多晚重复出现高风险记录、憋醒样声音、强烈喘息或用户明显不适时，建议用户带记录咨询医生。
不要要求用户上传隐私数据。不要制造恐慌。
当前角色是：{role_name}
角色语气是：{role_style}
```

角色风格参数：

- 贴心小妹：活泼、亲近、轻松、像家人。
- 懂事小弟：认真、清楚、孝顺、条理化。
- 阳光小哥：可靠、行动导向、坚定但温柔。
- 温柔姐姐：耐心、成熟、稳定、默认温和。

## 10. 数据与设置

当前 APK 已有：

- `PreferenceStore.companionRole()`
- `PreferenceStore.setCompanionRole(String role)`
- 设置页角色选择。
- 晨间护理角色文案。

后续建议扩展：

- `assistant_voice_enabled`
- `assistant_voice_speed`
- `assistant_chat_online_enabled`
- `assistant_avatar_asset`
- `assistant_last_greeting_date`
- `assistant_proactive_morning_enabled`
- `assistant_medication_tone`

本地数据原则：

- 角色选择只保存在本机。
- 聊天记录默认不长期保存。
- 若保存聊天记录，必须提供一键删除。
- 大模型联网状态必须在界面上清楚显示。

## 11. 界面设计

### 11.1 我的助手设置页

内容：

- 当前角色大头像。
- 四个角色卡片。
- 每个角色一句说明。
- “试听一句”按钮。
- “设为我的助手”按钮。
- “联网聊天增强”开关，默认关闭。

卡片文案：

- 贴心小妹：活泼温柔，像家人一样陪你。
- 懂事小弟：认真清楚，帮你记事和提醒。
- 阳光小哥：可靠坚定，适合安全守护。
- 温柔姐姐：温和耐心，默认陪伴助手。

### 11.2 首页入口

首页不做复杂聊天窗口，只显示：

- 小头像。
- 一句短话：“今晚我会静静守护。”
- 按钮：和我聊聊。

夜间守护中：

- 小助手保持静态。
- 不播放动效。
- 不主动说话。

### 11.3 晨间护理页

顶部：

- 大头像。
- “早安，昨晚辛苦了。”
- 语音播放按钮。

卡片：

- 昨晚汇报。
- 喝水。
- 吃药。
- 晨练。
- 天气小贴士。
- 和我聊聊。

### 11.4 强唤醒页

小助手只作为辅助，不喧宾夺主：

- 大标题仍是“请醒一下”。
- 大按钮仍是“我没事，继续守护”。
- 角色语音可播放，但亲人录音优先。
- 不展示复杂头像动画，避免分散注意力。

## 12. 验收标准

角色验收：

- 四个角色能在设置页选择。
- 切换角色不影响检测阈值、唤醒策略和联系人升级。
- 晨间护理文案会随角色变化。
- 强唤醒语音文案会随角色变化。

UI 验收：

- 头像不遮挡文字。
- 大字体在小屏手机不溢出。
- 角色卡片一屏内可清楚选择。
- 中老年用户能在 3 次点击内切换助手。

安全验收：

- 夜间守护不因小助手产生提示音。
- 小助手不主动诊断。
- 大模型默认关闭。
- 联网增强开启前有明确隐私说明。

内容验收：

- 每个角色至少 10 条基础语音文案。
- 每条建议都有依据。
- 异常频繁时建议咨询医生，而不是给诊断结论。
