## 已实现功能

- 原生 Android APK。
- 不注册、不收费；小助手主流程转为联网 AI，用户配置 Key 后调用 DeepSeek。
- 首次引导：直接开始 / 简单设置。
- 大字体首页：开始守护、停止守护、睡前检查、昨晚摘要。
- 前台守护服务：麦克风和加速度计本地采样。
- 静音守护通知：守护运行只显示静音常驻通知，不发运行提示音、不震动。
- 守护心跳：前台服务定期写入本地心跳，首页和睡前自检显示是否刚刚活跃。
- 睡前自检：检查权限、静音通知、唤醒声音、紧急联系人，并可测试轻提醒/强唤醒；手动测试强唤醒为演练模式，不会真实打电话或发短信。
- 电池优化引导：睡前自检会提示是否已允许不受电池优化限制，并可跳转系统设置。
- 守护完整性评分：基于麦克风、通知、电池优化、守护心跳、紧急联系人给出 0-100 分。
- 检测可信度说明：明确当前是声音/动作强度 + 本机个人基线，不是医疗诊断。
- 个人基线学习：守护平稳窗口会在本机学习安静声音和动作基线，动态调整中风险/高风险阈值，并在事件证据中写明当时阈值。
- 检测测试页：可模拟鼾声只记录、咳嗽/轻喘轻提醒、尖叫/强动作强唤醒，并生成测试 WAV 片段用于验收播放流程。
- 晨间护理：用户点击“我醒了，早安护理”后停止守护，展示睡眠汇报、喝水提醒、晨练小贴士和天气小贴士。
- 清晨低打扰确认：守护足够久且处于 05:30-11:00 清晨窗口时，若检测到明显动作或守护超过约 7 小时，会发静音“早安，醒了吗？”通知；用户点开后进入晨间护理，不点开则继续守护。
- 吃药提醒：用户主动设置药名/备注和重复间隔后，可确认“已吃药”或稍后再次提醒。
- 小助手体系：可选择贴心小妹、懂事小弟、阳光小哥、温柔姐姐；首页显示当前助手头像占位、角色语气和“和我聊聊”入口。
- 我的助手设置页：四个角色卡片、角色说明、示例话术、系统 TTS 试听、联网陪伴开关和 Key/模型设置。
- 自然语言第一次认识：不单开认识页；聊天页里小助手先礼貌打招呼，主人可直接说“以后叫你暖暖，像女儿一样陪我”，系统从这句话里记住名字和身份。
- 主人档案：可记录身体状况、用药习惯、睡眠情况、家庭情况、兴趣爱好和关怀偏好；支持小助手分步建档，用于联网建议上下文，不做诊断。
- 今日状态记忆：小助手可记录当天心情、精力和身体备注，用于当天关怀建议、早安护理和 DeepSeek 主动提问上下文。
- 主动关怀：用户开启后，App 打开时每天最多主动问候一次；夜间守护中不打扰。
- 陪伴人格：小助手系统提示词加入“听话顺从、礼貌乖巧、富有同情心、乐于倾听、先理解不争辩、哄老人开心”的规则；涉及药量、急症和危险事项时仍温柔劝阻。
- 灵动聊天界面：小助手头像占位加入轻微呼吸/摇摆动画，聊天页显示档案摘要、今天状态和快捷陪伴任务，老人端不大面积显示“AI”字样。
- 聊天页：可一键让小助手帮忙看昨晚、安排今天、陪用户聊天，也保留基础固定话术作为兜底。
- 摄像头权限前置：首次直接开始或完成简单设置后，会和麦克风/通知一起申请摄像头权限，方便小助手聊天时直接看见。
- 自动视觉陪伴：进入小助手聊天页后默认启动一次原生 Camera2 低清采样，图片缩放并压缩到约 220KB 内再分析；不在夜间守护和后台偷拍，可在聊天页暂停。
- 视觉陪伴：聊天页保留“让小助手看一眼”，用户主动授权摄像头后可拍照看气色、看药盒/药杯、读书/说明书、看外面和找东西；照片写入 App 缓存而不是相册。
- 位置记忆数据库：小助手会默认识别并记录钥匙、手机、药盒/药瓶、眼镜、钱包、证件/医保卡、遥控器、拐杖、水杯等常见物品位置，写入本机 `object_memory` 表；下次再看到同一物品会更新最近位置。
- 找物回答：用户问“我的钥匙在哪/丢哪了/找不到了”时，先走本机位置记忆回答最近看到的位置，例如“我记得昨天下午 16:00 看见钥匙在冰箱门上的挂钩，您先去那里看看还在不在。”
- 吃药看见记录：拍照后仍需用户主动确认“今天已吃药”，系统会记录确认并避免重复提醒；不根据照片判断药量、换药或停药。
- 联网看图通道：已接入主动拍照后的图片请求格式，配置支持图片输入的模型后可让小助手看图朗读、找物和给生活提醒；失败时保留本机兜底。
- DeepSeek 联网陪伴：用户可在手机本机设置 DeepSeek API Key 和模型；Key 使用 Android Keystore 加密保存，主动提问或点击快捷陪伴任务时联网，夜间强唤醒保留本地兜底。
- DeepSeek 本机测试注入：`android-app/local.deepseek.properties` 保存测试 Key，已加入 `.gitignore`，不会被 `build.ps1` 复制进 APK；连接测试机后可运行 `android-app/set-deepseek-test-key.ps1` 写入 App 本机设置。
- 角色化强唤醒：强唤醒页会使用当前小助手的确认语气，但不改变检测阈值、强唤醒策略和紧急联系人升级。
- 本地事件数据库：SQLite 保存事件、反馈、汇总。
- 复盘摘要：昨晚记录、7 天记录、高风险计数，可作为小助手生成建议的结构化上下文。
- 异常证据包：每条疑似异常保存触发依据、手机麦克风 RMS/峰值、加速度计动作评分、外部设备摘要状态。
- 现场录音片段：用户开启时，疑似异常会保存触发前一段本机 WAV 短录音，可在记录页播放；不保存整夜录音。
- 外部设备摘要：设置页可填写手表、手环、血氧仪等摘要，事件记录会明确显示已接入或未接入。
- 结构化设备读数：可手动录入心率、血氧、呼吸率和备注，系统会把异常事件与前后 15 分钟内最近的设备读数对齐。
- 复盘判断：记录页提供“复盘判断”弹窗，按现场录音、动作评分、手表/血氧仪摘要和多晚重复情况指导用户标记真实/误报/不确定。
- 中风险轻提醒：震动并记录自动取消。
- 高风险强唤醒：全屏页面、响铃、震动、大按钮确认。
- 语音停止尝试：强唤醒期间尝试本地/系统离线语音识别“我没事/停止”。
- 音量键确认：强唤醒时可用音量键确认。
- 紧急联系人：本机保存最多 3 位联系人，支持短信通知全部联系人，并拨打第 1 位联系人升级。
- 蓝牙音箱：通过系统音频路由使用，设置页可打开系统蓝牙设置。
- 唤醒声音：支持亲人录音、本地歌曲选择、试听、恢复系统闹钟。
- 强唤醒音源：优先播放亲人录音或本地歌曲，失败后回退系统闹钟。
- 导出证据摘要：通过系统分享面板导出给自己、家人或医生复盘的文本摘要，默认不附带音频。
- 本地删除：可删除事件、汇总数据和异常现场录音片段。

## 需要真机验收的点

- Android 13+ 通知权限弹窗。
- 摄像头权限弹窗、拍照返回缩略图、不同厂商相机 App 的兼容性。
- 原生 Camera2 自动低清采样在不同机型上的前后摄像头选择、低光画面和资源释放。
- 支持图片输入的联网模型是否能稳定识别药盒、书页、外部环境和低光画面。
- Android 14+ 麦克风前台服务限制。
- 锁屏强唤醒是否能弹出全屏页面。
- 各厂商电池优化是否杀前台服务。
- 电池优化白名单请求在不同厂商系统上的表现。
- Android 12+ 精确闹钟权限和各厂商提醒可靠性。
- CALL_PHONE / SEND_SMS 权限和实际拨打/短信行为。
- 系统语音识别在无网络时是否有离线包。
- 蓝牙音箱在不同手机上的音频路由稳定性。
- 清晨低打扰通知在锁屏、勿扰和厂商后台策略下是否能正常出现，且不发提示音、不震动。

## 当前限制

- 首版算法是规则引擎，不是医疗级模型。
- 当前检测不能可靠区分同床人、环境噪声和用户本人声音；记录页提供证据供复盘，后续仍需加入音频事件模型和可穿戴原始数据自动接入。
- 视觉陪伴当前在聊天页自动单次采样或用户主动拍照触发，不做后台连续摄像头监控；真实看图质量取决于联网模型是否支持图片输入、光线、对焦和拍摄角度。
- 脸色观察只做关怀提示，不能判断疾病；吃药照片只做提醒记录，不能判断药量、换药或停药。
- 不保存整夜录音，不做疾病诊断。
- 亲人录音使用 3GP/AMR 本机文件保存；后续可升级为更高质量 AAC 并加入音量归一化。
- 本地歌曲通过系统文件选择器授权；如果授权被系统回收，会回退系统闹钟。
- 小助手当前使用原生动画头像占位和系统 TTS；真实 3D 头像和更自然的语音聊天尚未接入。
- DeepSeek Key 已使用 Android Keystore 加密保存；正式发布前仍需在目标 Android 机型上验证 Keystore 迁移、清除和调试注入流程。
- 当前调试包声明 `android:debuggable="true"`，用于本机注入测试配置；正式发行包应关闭 debuggable，并继续禁止把 Key 打进 APK。
- Google Play 对短信/电话权限有审核风险，如上架受限可切换到系统拨号页/短信编辑页回退。

## 验收建议

1. 安装 APK。
2. 首次打开选择“直接开始”。
3. 授权麦克风和通知。
4. 点击“测试强唤醒”，确认页面、铃声、震动和大按钮可用。
5. 设置 1-3 位紧急联系人，测试权限说明和保存。
6. 点击“开始守护”，锁屏 1-2 分钟，确认前台通知存在。
7. 制造较大声音或明显晃动手机，确认记录页生成事件并展示证据包。
8. 如已开启异常片段，点击“播放现场录音”确认本机短录音可播放。
9. 点击记录页反馈按钮，确认反馈保存。
