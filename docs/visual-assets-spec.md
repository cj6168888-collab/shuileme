# 狗熊睡眠视觉资产方案

## 品牌方向

关键词：温馨、可靠、可爱、明朗、亲切、适合家庭和中老年用户。

## Logo

概念：一只闭眼睡觉的圆润狗熊，抱着月亮或枕头，旁边有小星星。不要凶猛、不要写实、不要医疗感。

图形要求：

- 圆润轮廓。
- 大面积暖色。
- 小尺寸可识别。
- 可用于 Android 启动图标。

## 配色

- 清晨蓝：`#2F80ED`
- 薄荷绿：`#27AE60`
- 晨光橙：`#F2994A`
- 柔和红：`#EB5757`
- 暖白：`#FFFDF8`
- 深灰：`#243040`

## 组件风格

- 大按钮。
- 大圆角卡片。
- 图标配文字。
- 避免复杂波形和密集图表。
- 强唤醒页高对比、超大字。

## 首版实现

首版 APK 采用原生 View + 图片资产组合：核心页面直接按视觉参考板拆分出品牌 logo、首页睡眠场景、晨间护理场景、强唤醒闹钟和四个陪伴角色头像，保证安装包离线可用且视觉风格统一。后续新增正式页面或组件时，不再用临时占位图作为最终交付。

## 设计大模型工作流

当需要新增或升级这些视觉元素时，优先请绘图/设计大模型产出正式位图资产，再接入 APK：

- 页面背景：夜间守护、晨间护理、强唤醒、聊天陪伴、首次引导。
- 核心组件：开始守护按钮、我没事按钮、我醒了按钮、喝水/吃药/安全状态卡。
- 状态插画：守护已就绪、轻提醒、自动取消、唤醒失败、紧急通知。
- 角色资产：贴心小妹、懂事小弟、阳光小哥、温柔姐姐的头像、表情和动效帧。
- 商店物料：应用图标、宣传横幅、截图背景、权限说明插图。

产出要求：

- 使用原创暖色 3D 狗熊睡眠视觉语言，不模仿真实明星、版权角色或具体商业 IP。
- 输出 PNG/WebP 位图，移动端 2x/3x 清晰；透明组件优先 PNG，整屏背景优先 WebP/PNG。
- 同一批资产保持统一光照、镜头角度、圆角、阴影和色彩饱和度。
- 按 Android 用途切片保存到 `drawable-nodpi` 或对应密度目录，源图和提示词留存在 `docs/assets`。
- 接入后必须在模拟器截图验收，确认字体、按钮、图片不遮挡，长辈模式字号下仍然可用。

## 视觉概念图

- 视觉参考板：[`docs/assets/gouxiong-sleep-ui-brand-board.png`](assets/gouxiong-sleep-ui-brand-board.png)
- 小助手灵动聊天参考板：[`docs/assets/companion-live-chat-ui-board.png`](assets/companion-live-chat-ui-board.png)

这张图现在作为原生 APK 的主视觉基准：首页、早安护理、强唤醒和小助手角色卡按这套暖色、3D 狗熊、大按钮风格实现。

可采用：

- 温暖 3D 狗熊 logo。
- 暖白背景。
- 蓝色主按钮。
- 绿色安全状态。
- 橙色晨间护理。
- 红色强唤醒。
- 大字体、大按钮、大卡片。
- 四个陪伴角色方向：贴心小妹、懂事小弟、阳光小哥、温柔姐姐。
- 小助手聊天页需要“像活的”：头像有呼吸、眨眼、挥手、点头等轻动画；主动关怀卡片应柔和浮入，不用压迫式弹窗。

需要修正：

- 概念图里的部分文案只是视觉占位，最终 APK 文案必须以产品文档为准。
- “我没事”只用于强唤醒确认，不用于喝水卡片。
- “起床啦”可用于晨间护理，不用于夜间强唤醒。
- 陪伴角色名称可以亲切，但不能暗示医疗诊断或治疗。

## 可开发资产拆分

后续应拆成这些独立素材：

1. Android 启动图标：狗熊抱月亮，圆形安全区。
2. 首页吉祥物：睡觉狗熊，适合深夜首页。
3. 晨间护理吉祥物：醒来挥手狗熊，适合早安页。
4. 强唤醒插图：闹钟或亮色提醒，不使用惊吓画面。
5. 陪伴角色头像：四个 3D 半身头像，统一光照和背景。
6. 应用商店横幅：强调“离线守护、晨间护理、家人安心”。
7. 聊天页动效帧：呼吸、眨眼、挥手、点头、倾听、提醒六种状态。

## 角色设计要求

- 小女孩：活泼、温柔、可爱，适合晨间问候和轻松聊天。
- 小男孩：聪明、孝顺、可爱，适合提醒和陪伴。
- 男青年：阳光、温柔、可靠，适合安全提醒和报告讲解。
- 女青年：贴心、温暖、耐心，适合健康生活建议。

角色都要：

- 看向用户，适合面对面聊天。
- 表情亲切，不夸张。
- 服装日常、干净、不过度职业医疗化。
- 不使用白大褂，避免医疗诊断暗示。
- 不使用真实明星或版权角色风格。
