# 狗熊睡眠 UI/UX 对照归类审查

版本：V1.0  
依据：用户提供的温馨狗熊睡眠参考图、现有 APK 模拟器截图、`docs/independent-ui-ux-redesign-plan.md`。

## 1. 组件归类

| 类别 | 参考图职责 | 当前问题 | 调整方向 |
| --- | --- | --- | --- |
| 品牌与吉祥物 | 一眼识别“狗熊睡眠”，温暖可信 | 首页大图仍像手机截图，带内嵌状态栏和标题 | 首页先用品牌/吉祥物独立区，后续替换为 `gxs_*` 独立资产 |
| 主行动按钮 | 蓝色大按钮，只有一个主动作 | 首页下方重复出现多颗大按钮，第一屏决策负担重 | 第一屏只保留“开始/停止守护”，其它动作进宫格 |
| 状态确认 | 绿色“守护已就绪”清楚表达安全 | 检查说明太长，像工程报告 | 改成 3 个短胶囊：麦克风、音量/蓝牙、联系人 |
| 快捷宫格 | 安心守护、晨间关怀、家人陪伴、离线/记录 | 宫格基本正确，但“睡眠记录”图标像占位 | 保留结构，后续换独立图标资产 |
| 早安护理 | 明亮、少字、喝水/吃药可执行 | 早安页堆叠过多说明卡 | 顶部问候 + 喝水/吃药大卡优先，报告下沉 |
| 强唤醒 | 超大确认按钮，紧急但不吓人 | 插图含内嵌文字和状态栏，按钮文案有重复 | 后续拆成闹钟主体、背景、原生标题和按钮 |
| 小助手 Live | 面对面、能打断、少按钮 | 仍有“实时陪伴”说明卡，角色图片被白框包住 | 去掉图片卡片感，保留三个核心动作 |
| 设置/档案 | 低频、可滚动、清楚分组 | 大量设置按钮平铺 | 后续分组为“安全、声音、家人、小助手、数据” |

## 2. 目前最不合理的点

1. 首页首图不是独立视觉资产，仍像把整台手机界面截图放进 App。
2. 第一屏信息密度过高，既有守护状态，又有算法可信度、昨晚摘要、多个重复按钮。
3. 图片控件自带卡片背景，导致小助手大图、场景图出现“卡片套卡片”的廉价感。
4. `ui_*` 过渡素材仍承担正式视觉职责，和“绘图模型逐组件设计”的要求冲突。
5. 早安页、强唤醒页的图里含文字，应该由 Android 原生大字渲染。

## 3. 已在本轮先修

- 首页重排为：品牌视觉区、守护状态、开始守护、三项就绪胶囊、四宫格、睡眠情况。
- 删除首页重复大按钮，把“睡前自检、早安护理、强唤醒测试”从主流程降级到宫格或后续区域。
- 早安页撤掉带状态栏和内嵌标题的旧图片，改成原生“早安护理”标题和晨间动作卡。
- 喝水动作改为“我喝水了”，“我没事”只保留给异常唤醒确认。
- 图片控件不再自动套白色卡片和阴影，减少截图感。
- 小助手 Live 大图不再被额外卡片框包住。

## 5. 验收标准

- 第一屏只出现一个主行动。
- 图片中不能出现 Android 状态栏、按钮、最终 UI 文案。
- “我没事”只用于唤醒确认，不用于喝水普通卡片。
- 检测可信度、算法边界、报告类长文本不能压在首页第一屏。
- 小助手页打开即像面对面聊天，不像表单或设置页。
