# 狗熊睡眠检测算法与测试方案

目标：用手机本地规则先做可解释、可调试、可离线的异常识别。

## 检测输入

- 麦克风：短窗口 RMS、峰值、静默时长、近似周期性。
- 加速度计：动作强度、突发动作、手机位置变化。
- 用户反馈：真实、误报、不确定。
- 可穿戴：后续接入 Health Connect 后再加入心率、血氧、呼吸率摘要。

## MVP 规则

当前 APK 采用“固定安全底线 + 本机个人基线”的规则：平稳窗口会学习安静声音 RMS 和动作评分，样本达到 20 个后首页显示“个人基线已启用”。中风险/高风险阈值会随基线在安全范围内动态调整，明显高强度声音或强动作仍保留固定兜底触发。

### 声音事件

- 普通环境：低 RMS、短峰值、无持续异常。
- 鼾声候选：中等 RMS、周期性出现、持续多个窗口。
- 喘息/呛咳候选：静默后短促高峰。
- 尖叫/惊恐候选：高 RMS、高峰值、持续超过阈值。
- 咳嗽候选：短促高峰、间隔重复。

### 动作事件

- 翻身：中等动作强度，持续短。
- 强动作：动作峰值明显超过个人基线。
- 手机移动：持续姿态变化，降低本晚可信度。

## 风险分级

- 低风险：只记录。
- 中风险：轻提醒，随后观察是否恢复平稳。
- 高风险：强唤醒。
- 升级风险：强唤醒无确认，触发电话/短信。

## 自动取消

轻提醒后 10-30 秒检查：

- 无持续尖叫。
- 无连续喘息/呛咳候选。
- 动作强度回落。
- 声音节律恢复平稳。

满足则记录为自动取消。

## 测试素材

开发测试至少准备：

- 鼾声音频。
- 咳嗽音频。
- 喘息/呛咳音频。
- 尖叫/喊声音频。
- 梦话/普通说话音频。
- 环境噪声。
- 静音片段。

## 验收指标

- 测试音频触发事件后，3 秒内进入相应状态。
- 平稳环境运行一段时间后，首页检测可信度显示个人基线样本数增加。
- 事件记录应写入当时声音/动作动态阈值，方便复盘误报和漏报。
- 中风险轻提醒后，平稳窗口能自动取消。
- 高风险能打开强唤醒页。
- 误报反馈能保存到数据库。
- 断网状态下仍能生成事件和汇总。

## 当前 APK 内置测试机制

- `模拟鼾声：只记录`：写入低风险记录，不唤醒。
- `模拟咳嗽/轻喘：轻提醒并自动取消`：写入中风险记录并震动。
- `模拟尖叫/强动作：强唤醒`：写入高风险记录并打开强唤醒页。

这些是链路测试，不代表真实识别准确率。真实识别测试仍需外部测试音频在真机环境播放。

## 真实音频测试集要求

每类样本至少 20 段，覆盖近距离、远距离、同房噪声和不同音量：

- 鼾声。
- 咳嗽。
- 喘息/呛咳。
- 尖叫/喊声。
- 梦话/普通说话。
- 环境噪声。
- 静音/风扇白噪声。

每段测试要记录：是否触发、触发延迟、事件类型、是否误报、是否需要调整阈值。

## 非诊断边界

算法只输出“疑似事件、风险等级、触发依据”，不输出疾病诊断。
